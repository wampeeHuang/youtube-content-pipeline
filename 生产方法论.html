00任务书：项目定义 · 张咋啦

任务书定义完整项目需求——做什么、交付什么、怎么管、怎么判断做对了。后续管线全部服务于这份任务书。

0.1 项目动机

为 B站科技区观众提供高质量中文字幕的海外 AI 访谈搬运。消除语言障碍，保留原汁原味——不做 AI 配音、不做二次创作、不做观点嫁接。

定位：AI 辅助 + 人工品控的精翻搬运。差异化：精翻品质（非机翻）、精选嘉宾（非新闻批量）、完整管线可复现、封面/标题适配 B站审美。

0.3 管线全流程（需求层级）

管线是满足任务书的手段——任务书回答"要什么"，管线回答"怎么做"。完整实现见 §03。

① 选题→ ② 下载→ ③ 断句→ ④ 翻译→ ⑤ 判定→ ⑥ ASS→ ⑦ 渲染→ ⑧ 金句提取→ ⑨ 封面→ ⑩ 标题→ ⑪ 元数据→ ⑫ 专栏→ ⑬ 发布

关键关口：④ 翻译是素材质量判定点——翻译完成后才能判断内容价值，决定继续或排除（见 §01 影视飓风差异）。⑤ 判定是 Go/No-Go 决策点。⑧ 金句提取决定封面和标题的钩子质量。

0.4 服务对象 & 质量标准

#观众期待我们的对应
1翻译准确、自然，不是机翻味AI 精翻 + 人工抽检，赞助商口播等低价值段落排除
2字幕不遮挡、不闪烁、中英同步中英双语同屏同步显示，阅读节奏舒适
3封面一眼看出嘉宾和核心观点嘉宾正脸 + 核心金句，AI 多方案择优，信息流中可辨识
4标题不标题党，忠实传达内容嘉宾身份+核心论断格式，禁用【】/emoji
5内容是B站上没有的独家观点每期 B站交叉检索，已有精翻的放弃

平台硬约束（转载红线、数据规格、竞品格局、投稿合规）→ 见 §05.2 平台规范速查。

0.9 反馈闭环（数据 → 选题）

每期发布后 24h / 7d / 30d 记录数据，反馈到下一轮选题决策。不记录就没有改进。

指标采集时点反馈什么
播放量24h / 7d / 30d嘉宾热度 → 同类嘉宾调高优先级；低于预期的分析原因
点击率（播放/曝光）7d封面+标题效果 → 调整金句提取策略和封面设计
留存率 / 平均播放时长7d内容质量 → 调高/调低内容源优先级；短留存 = 内容不匹配
互动（点赞/收藏/评论）7d / 30d社区认可度 → 长期价值维度的验证
粉丝增长30d频道健康度 → 选题方向是否正确

反馈路径：数据记录到 CSV（候选池行 + 已发布/slug.md）→ 定期回顾 → 调整评分权重、内容源优先级、封面/标题策略。

01案例分析 · 潘乱

三个案例，三个视角：影视飓风（原创方法论，差异中找可借鉴）、AI译片君（赛道参照，借鉴/不借鉴）、Kevin Weil 期（自评复盘，调优基线）。

1.1 影视飓风：原创方法论 vs 搬运的差异

B站 900 万+粉丝，科技区标杆。2025 年首次公开十年创作方法论。

壳理论（对我们最重要的一条）

影视飓风：先定标题和封面，再定内容架构——用生活化钩子（壳）包裹硬核知识（核）。
例：《泰勒演唱会为什么没有摩尔纹？》400 万+播放 vs"什么是摩尔纹"无人问津。

搬运的差异：我们内容已存在，不能"先定标题再拍"。流程是反过来——先翻译才能判断素材质量。
可借鉴：翻译完成后，从 transcript 中提取最精彩的金句作为"壳"，倒推设计封面和标题。壳不是编的，是从已有内容中选的。Kevin Weil 期用的就是这套——"你现在用的 AI 是你余生最差的"= 壳，Kevin Weil 的完整访谈 = 核。

HKRR 四维 → 我们的质量门禁

维度影视飓风我们的适配
Happiness 快乐让观众感到愉悦搬运不创造快乐，但可以选择信息密度高、不枯燥的内容
Knowledge 知识提供信息增量核心维度。翻译后判定：有没有新观点/新数据/新框架？不过关 → 排除
Resonance 共鸣情绪层面的连接嘉宾的坦诚度、反直觉观点、行业内幕 → 观众"原来如此"感
Rhythm 节奏信息密度的控制赞助商口播 = 节奏断裂点，需检测并跳过。断句参数控制阅读节奏

其他可借鉴原则

原则影视飓风做法我们怎么用
放弃内部梗每多一个内部梗，就限制一波受众不用「双语字幕」「熟肉」「烤肉」标签，默认品质不挂标签
AI 多方案择优AI 生 20+ 封面方案，5 分钟初筛封面不是"做一张"，是"生成一批挑最好的"（§02）
质量 > 频率45 天出一期 15 分钟内容宁可一周一期精的，不要一天一期糙的
先定差异化锚点把最有差异化的点放开头封面大字放最有冲击力的金句（不是概括，是钩子）

1.2 AI译片君：赛道参照

AI 翻译+语音合成，批量产出，27 万粉 / 3.2 亿播放。B站 AI 搬运赛道最大玩家。

借鉴什么

  • 深色封面 + 暖色大字 — 已验证的 B站科技区最优配色
  • 稳定更新频率 — 习惯性订阅靠的是产出节奏
  • 标题格式化 — 嘉宾身份 + 核心观点，清晰可预期

不借鉴什么

  • AI 配音覆盖原声 — 我们去掉了原汁原味的最大优势
  • 批量机翻品质感低 — 观众能分辨，不挂"双语字幕"标签的原因
  • 早期同类号已被处罚 — 合规风险：转载标注必须清晰
  • 新闻搬运长期价值低 — 不做日更/纯新闻

02封面 & 标题 · 李晨阳

封面和标题是 B站信息流中同时展示的两个钩子，各承担 50% 的点击驱动。影视飓风方法论：先定壳（封面+标题），再裹核（内容）。我们的适配版：翻译完成后从 transcript 提取最佳金句做壳。

2.1 封面设计系统

色彩策略

深色底 + 暖色大字（#FFC82D）+ 人物正脸 = B站科技访谈类最优解。白色信息流中深色封面天然跳出。不采用白底极简（淹没在信息流中）。

字体层级（1920×1080 画布）

层级字体字号颜色字数功能
L1 · 封面大字SimHei + 2px 黑填充140–180px暖黄 #FFC82D4–8 字钩子，一眼可读
L2 · 副标题SimHei60–80px暖白 #FCFAF510–16 字补充上下文
L3 · 出处行SimHei24–28px灰 #8C8275不限底部署名
Windows PIL 只有 SimHei 够粗。2px 四周黑填充模拟 Bold→Heavy 加粗效果。

四种布局

2.3 封面与标题的协同

3.2 下载

yt-dlp --write-auto-subs --sub-langs en --convert-subs srt \
-f "bestvideo[height<=1080][vcodec^=avc1]+bestaudio[ext=m4a]/..." \
--output "%(title)s.%(ext)s" <url>
必须 H.264(avc1)+AAC(m4a)，否则拿到 webm/vp9 — B站不兼容。

3.3 断句 → 翻译 → ASS → 渲染

环节关键参数
断句~18 英文词/句，句尾标点切割，硬切上限 27 词，carryover 只裁文本不移时间戳
翻译DeepSeek Chat, temp=0.3, 10条/批, 并发5, 超时120s, 重试3次退避1s/2s/4s
中文字数上限 28 字，拆分点 ，。！？；、，硬切兜底 56 字，时间按字数比例分配
ASS画布 1920×1080，中文 SimHei 42px，英文 Segoe UI 32px Bold，\N 单事件双语，MarginV=45，Alignment=2
渲染ffmpeg -y -i video.mp4 -vf "ass=05.ass" -c:v libx264 -crf 23 -c:a aac -b:a 128k output.mp4

翻译 Prompt

System: You are a professional EN→ZH subtitle translator.
Output ONLY the Chinese translation for each line, one per line. No explanations.

User: Translate these English subtitles to Chinese (Simplified).
Keep it concise and natural. Preserve technical terms where appropriate.
ASS 为什么必须单事件 \N：libass 碰撞检测 — 两条相邻 Dialogue 时间重叠时自动隐藏一条。中英文随机消失。
函数签名：enforce_max_chars(srt_path, max_chars=28) — 不是 (srt_path, output_path)
