排版形式（Layout Patterns）

基于 B站科技访谈类高播放封面采样，归纳 4 种主流布局

布局 A：人物左 + 文字右（最主流）
┌──────────────────────────────────────┐ │ ┌──────────┐ │ │ │ │ 主标题大字（4-8字） │ │ │ 人物 │ 暖黄 #FFC82D │ │ │ 正脸 │ ─── 装饰线 ─── │ │ │ 占40% │ 副标题小字 │ │ │ │ 出处行 │ │ └──────────┘ │ └──────────────────────────────────────┘

适用：人物半身照可清晰截取时。B站 "OpenAI 访谈" 搜索结果中高播放封面 80% 用此布局。人物视线应朝向文字方向。

布局 B：全幅底图 + 文字叠层
┌──────────────────────────────────────┐ │ ╔══════════════════════════════════╗ │ │ ║ [半透明黑底条 或 渐变遮罩] ║ │ │ ║ 主标题大字（居中） ║ │ │ ║ 副标题 ║ │ │ ╚══════════════════════════════════╝ │ │ [底图：人物/场景，亮度 0.6–0.8] │ │ │ │ 出处行 │ └──────────────────────────────────────┘

适用：底图本身信息量高（如两人对谈、舞台场景）。文字区需要半透明黑底衬托。AI译片君大量使用此布局。

布局 C：居中对称
┌──────────────────────────────────────┐ │ │ │ 主标题大字（居中） │ │ ─── 装饰线 ─── │ │ 副标题 │ │ │ │ [人物居中，半身像] │ │ │ │ 出处行 │ └──────────────────────────────────────┘

适用：人物在画面正中央，文字在上方。正式感强，适合单人演讲截图。注意文字不能遮挡人脸。

布局 D：上下分割
┌──────────────────────────────────────┐ │ ▓▓▓▓▓▓ 深色背景区 ▓▓▓▓▓▓▓▓▓▓▓▓▓ │ │ 主标题大字 + 副标题 │ │ ░░░░░░ 图片区 ░░░░░░░░░░░░░░░░░░░ │ │ [人物 / 场景截图的另一半] │ │ 出处行 │ └──────────────────────────────────────┘

适用：底图不够大或不适合叠字时。上 30-40% 纯色区放文字，下 60-70% 放图片。YouTube 科技频道常用此风格。

排版通用原则
  • 视觉焦点唯一：每张封面只有一个核心信息锚点（金句），其余为辅助
  • 文字非说明是钩子：封面文字的功能是「吸引点击」，不是「概括内容」
  • 人物面部 ≥ 画面 25%：表情和视线是点击率的核心驱动
  • 九宫格交点放核心元素：主体（人脸/主标题）落在四个交点之一
  • 构建三层亮度结构：亮主体（RGB 180–240）→ 中灰过渡（80–150）→ 深底色（≤50）
  • 不做信息过载：B站缩略图仅 ~200×120px，多余元素不可辨识 = 视觉噪音

搬运视频封面本土化要诀

  • 不直接用原视频封面：海外审美（白底/极简/英文为主）直接搬迁水土不服
  • 提取金句做中文大字：不是翻译原标题，是从内容中提炼最有冲击力的中文金句
  • 人物是信任资产：嘉宾正脸比产品图/抽象图点击率高 2-3 倍（行业数据）
  • 品牌锚点一致：固定暖黄 + 固定字体 + 固定底部出处行 = 频道视觉资产
  • 不加「双语字幕」标签：高质量字幕是默认配置不是卖点，挂标签反而像低质机翻号

0B.3 标题规范

参数标准
总字数≤ 80 字（B站硬限制）
推荐字数30–50 字（信息流完整展示）
语言中文为主，人名保留英文原名
禁止【】方括号、《》书名号、#话题标签、emoji 堆叠、??? / !!! 堆叠、繁体字、特殊符号
格式嘉宾身份 嘉宾名：核心论断

高点击标题套路（科技区实测有效）

  • 数字 + 结果：「3 分钟看懂 XXX」
  • 反常识 + 悬念：「你现在用的 AI 是你余生最差的」← 我们 Kevin Weil 期用的
  • 权威 + 独家：「OpenAI CPO 首次透露」
  • 对比 + 冲突：「国产 VS 海外 差距有多大」
  • 紧迫感：「全网首发」「别再用了」
我们不需要「双语字幕」后缀。高质量双语字幕是我们的默认配置，不需要在标题中声明。低质搬运号才挂这类标签。

我们的定位

AI 辅助 + 人工品控的高质量双语字幕搬运。
差异化：① 人工精翻品质（非机翻），② 精选嘉宾/深度访谈（非新闻批量），③ 完整管线可复现（非一次性），④ 封面/标题本地化适配 B站审美。
不做什么：① 不做 AI 配音（保持原声），② 不做日更/批量（质量优先），③ 不做纯新闻搬运（长期价值低）。

风险监控

  • AI译片君模式风险：早期类似账号（牢君、牢斑马）已被处罚/封禁，原因是违规搬运 + 商业化使用
  • 版权收紧趋势：B站对未授权搬运的审核在持续收紧
  • 我们的优势：精选内容 + 人工品控 + 转载标注，合规风险远低于 AI 批量号

布局模式选择

根据底图人物位置和画面信息量，选择对应布局：

条件推荐布局理由

字体层级

层级字体字号颜色字数功能
L1 · 封面大字SimHei + 2px 黑填充140–180px暖黄 #FFC82D4–8 字钩子，一眼可读

字号原则：L1 封面大字 = 画布高度的 13–17%。B站缩略图 ~200×120px 下，140px 以下字不可辨认。L2 字号为 L1 的 40-50%，保持层级跳跃明显（不是微差）。

视觉层级构建（三层亮度结构）

层亮度范围元素作用
前景 · 亮层RGB 180–255主标题文字（暖黄 #FFC82D ≈ RGB 255,200,45）视觉焦点，最先被看到
中景 · 灰层RGB 80–160底图人物面部、副标题文字提供内容和情感连接
背景 · 暗层RGB ≤ 60底图暗区、半透明黑衬底承载亮层，制造对比

构建方法：① 底图先调亮度至 0.80（让暗区更暗、亮区不过曝）→ ② 加半透明黑底条（rgba 0,0,0,0.5）覆盖文字区 → ③ 暖黄文字置于最上层。三层分离明显时，缩略图下也能一眼看到文字。

标题与封面的协同

封面和标题是 B站信息流中同时展示的两个元素，必须协同设计，不是各自独立。

策略封面文字标题后半段效果
封面抛钩 + 标题展开 「AI 每两个月就颠覆一次」 「OpenAI CPO Kevin Weil：你现在用的 AI 是你余生最差的」 最佳 封面钩子先抓眼球，标题给完整上下文
封面概括 + 标题重复 「Kevin Weil 访谈精华」 「Kevin Weil 深度访谈」 浪费 两个展示位说同一件事
封面空洞 + 标题全押 仅头像/Logo 超长标题塞全部信息 低效 封面在信息流中先被看到，封面没钩子=流失
原则：封面和标题各承担 50% 的钩子功能。封面用 4-8 字金句制造好奇心，标题用完整句式提供权威背书和上下文。两者互补不重复。
