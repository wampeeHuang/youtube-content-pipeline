# 选题方法论

> 选题是管线的第一步。选错 = 全链路白跑。本文档是选题判定的唯一真相源。

## 完整流程

```
发现内容 → 初筛 → B站交叉检索 → 五维评分 → 入库候选池 → 排期生产
```

## §2 初筛（3分钟判断）

打开视频，快速跳3-5个时间点，判断以下4项。任意一项NO → 放弃。

| 检查项 | 标准 | 判断方法 |
|--------|------|---------|
| **语言** | 英文演讲，清晰可懂 | 跳3个时间点听10秒 |
| **字幕** | 有官方英文字幕或高质量auto | 打开CC看准确率 |
| **信息密度** | 不是纯娱乐/闲聊 | 看内容是否围绕一个核心观点展开 |
| **视觉可用** | 主讲人正脸清晰可截图 | 快速浏览缩略图时间线 |

**初筛通过后：** 记录 URL + 标题 + 嘉宾名 + 日期 → 进入 B站交叉检索。

## §3 B站交叉检索

**每期选题必须完成，不可跳过。** 这是独占性判定的唯一依据。

### 方法

1. `WebFetch` 打开 `search.bilibili.com/video?keyword=嘉宾名+标题关键词`
2. 搜索结果中查找：
   - 相同视频的中文翻译版？
   - 同一嘉宾的其他视频搬运？
3. 判定结果：

| B站状态 | 独占性评分 | 行动 |
|---------|-----------|------|
| 零中文搬运 | 3 分 | 独占，继续 |
| 有搬运但无中字/纯搬运无翻译 | 2 分 | 备选，价值高仍可做 |
| 已有精品双语翻译 | **直接排除** | 一票否决 |

### 搜索关键词模板

```
<嘉宾英文名> 中文
<嘉宾中文名/常用译名>
<演讲/访谈核心主题>
<活动名> <年份>
```

## §4 五维评分

| 维度 | 权重 | 3 分 | 2 分 | 1 分 |
|------|------|------|------|------|
| **时效性** | ×3 | 6个月内 | 1年内 | >1年 |
| **独占性** | ×3 | B站零中字 | 有搬运无中字 | 已有精品中字→直接排除 |
| **人物权威** | ×2 | Anthropic核心/CEO/学界领袖 | 知名创作者 | 素人 |
| **长期价值** | ×2 | 方法论/框架/哲学 | 趋势分析 | 纯新闻 |
| **字幕可用** | ×1 | 官方字幕 | 高质量auto | 需whisper |

**总分公式：**
```
总分 = 时效性×3 + 独占性×3 + 人物权威×2 + 长期价值×2 + 字幕可用×1
满分 = 3×3 + 3×3 + 3×2 + 3×2 + 3×1 = 33
```

| 分数 | 行动 |
|------|------|
| ≥25 | **立即做** — 排入生产队列 |
| 20–24 | **备选** — 选题库不够时启动 |
| <20 | **排除** — 不录入选题库 |

## §5 排除标准（一票否决）

以下任意条件满足 → 直接排除，不进入评分：

| # | 条件 | 原因 |
|---|------|------|
| 1 | B站已有精品双语翻译 | 零独占性 |
| 2 | 版权敏感（电影/音乐为主） | 合规红线 |
| 3 | 纯娱乐/标题党/营销号风格 | 与频道定位不符 |
| 4 | 无英文字幕 + 语音质量差（口音重/背景噪音） | whisper成本高，翻译不可控 |
| 5 | 翻译后主观判定搬运价值不足 | Cat Wu Tokyo 2026 先例 |

## §6 选题库

选题库是候选池的落盘位置。**选题库在产出目录，不在项目目录。**

```
D:\workspace\_output\猫波信号站\选题库\
├── CLAUDE.md           ← 操作规范
├── 内容源.md            ← 监控源清单（博主/栏目/会议/聚合站）
├── 候选池/
│   ├── 候选池.csv        ← 主表，固定列 sort/filter 对比
│   └── <slug>.md        ← 仅≥25分候选追加详情
├── 已发布/              ← 已发布 ← 从候选池移入
│   └── YYYYMMDD_slug.md
└── 已废弃/              ← 判定无价值 ← 从候选池移入
    └── YYYYMMDD_slug.md
```

### 候选池.csv（主表）

每个候选一行，固定列天然适合同维度对比：

`date, slug, 嘉宾, 标题, URL, 时效性, 独占性, 权威, 长期, 字幕, 总分, 状态, 来源`

- **状态**：`候选` | `已发布` | `排除` | `过期`
- **排除行不删**，来源列写原因。防重复评估
- 按总分降序排列

### 详情 MD（仅 ≥25 分）

高分候选追加快照 MD，放 B站检索结果和初筛笔记：

```markdown
# <嘉宾名>：<视频标题关键词>

| 字段 | 值 |
|------|-----|
| URL | https://youtube.com/watch?v=xxx |
| 嘉宾 | <名字> |
| 来源 | YouTube / <频道名> |
| 日期 | <视频发布日期> |
| 时长 | <mm:ss> |

## B站交叉检索
<搜索词 + 结果摘要>

## 初筛笔记
<信息密度/视觉可用/语言质量>
```

评分已在 CSV 中，详情 MD 不重复评分列。

操作规则见 `D:\workspace\_output\猫波信号站\选题库\CLAUDE.md`。

## §7 排期决策

当候选池中 ≥25 分的候选 > 1 个时，按以下优先级排期：

1. **时效性倒计时** — 已发布 N 个月的视频优先于刚发布的（越等越贬值）
2. **人物热度** — 嘉宾近期有新闻/争议/新产品发布的优先
3. **性别/话题多样性** — 避免连续两期同类型/同嘉宾

## §8 已发布后的清理

生产管线完成后 → 候选文件从 `候选池/` 移到 `已发布/`，追加发布信息：

```markdown
## 发布信息

| 字段 | 值 |
|------|-----|
| 发布日 | YYYY-MM-DD |
| B站 BV号 | BVxxxxxx |
| B站标题 | <最终标题> |
| 数据 | 播放/点赞/投币/收藏（7天后回填） |
```

---

## §9 废弃流程

判定无搬运价值 → 候选文件从 `候选池/` 移到 `已废弃/`，追加废弃原因：

```markdown
## 废弃原因

- **日期**: YYYY-MM-DD
- **原因**: <时效性过期 / 内容重叠 / 翻译后质量不达标 / 其他>
- **教训**: <下次选题如何避免同类问题>
```

同时：
- 删除项目 `_runtime/<slug>/`（如已开始）
- 技术踩坑保留在 `_ref/pitfalls.md`（与具体视频无关）
