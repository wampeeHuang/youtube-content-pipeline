🥈 Head of Claude Code: What Happens After Coding is Solved | Boris Cherny

推荐 · 4个月前 1h28m 2026-02-19 Lenny's Podcast #294

嘉宾：Boris Cherny，Claude Code 创建者 & 负责人（Anthropic）

主题：当编程被 AI 解决后，软件行业会发生什么。Boris 自 2025 年 11 月起没手写过一行代码，每天靠 AI 提交 10-30 个 PR。Claude Code 已占 GitHub 全部公开提交的 4%，DAU 月月翻倍。核心观点：少招人、多给 token、不要让 UI 框住模型——模型自己能找到更好的交互方式。

📋 核心论点 + 观众反馈

核心论点：

  • 编程已解决：自 2025 年 11 月起 100% 用 AI 写代码，每天 10-30 个 PR
  • 少招人、多给 token：Anthropic 内部理念——刻意低配团队，给无限 token 预算
  • 不框住模型：给模型 bare tools（bash、脚本）而非包装好的 UI，模型自己找到最佳交互
  • 离开又回来：短暂加入 Cursor 两周后回归 Anthropic——放不下安全使命
  • Cowork 10 天建成：用 Claude Code 自身构建的协作版 Claude Code
  • 软件工程师 → Builder：PM、设计师、工程师角色融合，人人都是混合型 Builder
  • 印刷术类比：AI 对编程的民主化 = 古腾堡对识字的民主化

🔥 观众反应：B站已有搬运（BV1kEcZzJEKG）但为英文原版无中字。跨国串门儿计划（中文播客）已做解读集。多个微信公众号引用 Boris 的「少招人、多给 token」论点。Lenny's 频道下该集播放/讨论量在 2026 年排前三。

🎯 推荐理由：Boris Cherny 是 Claude Code 之父——用户每天用的工具就是他造的。内容极前沿——「编程已解决」「软件工程师头衔活不过 2026」等论断引发全球讨论。4 个月前的访谈在 AI 领域仍属新鲜。与 #1 互补：#1 是 Cat Wu 讲产品现状与未来功能，#2 是 Boris 讲底层哲学和行业影响。

📋 筛选标准

✅ 入选标准（按权重排序）

①时效性 ⏱️ — 发布日期是否在近 6 个月内？AI 领域迭代极快，2025 年中的观点到 2026 年中可能已被推翻。超过 1 年的内容需额外论证「为什么现在仍然值得翻译」。
②前沿性 / 启发性 — 是不是在讲还没成为常识的东西？有没有新观点、新框架、新数据？
③人物权威性 — 创作者/嘉宾在领域内有没有公认地位？（Anthropic 员工、CEO、学界大牛 > 素人播主）
④内容长期价值 — 非纯新闻速报，方法论和思想框架不随时间贬值
⑤B站独占性 — B站未出现高质量中字版本，搬运后具有信息增量
⑥英文原版字幕质量 — 有官方/高质量字幕优先（降低翻译校对成本）
⑦YouTube 播放量 > 10K（基础参考，不是硬门槛）

🚫 排除标准

①纯娱乐/标题党内容
②B站已有精品双语翻译版本
③版权敏感内容（电影/音乐为主）
④无英文字幕且语音质量差（whisper 转录成本高）
优先级公式： 独占性（B站无）× 内容长期价值 × 字幕可用性 × 与用户领域相关性 → 高/中/低三档。高 = 三个条件全满足且独占性确认；中 = 满足大部分但某维度稍弱；低/排除 = B站已有精品版本。

全链路流程

1
发现 & 筛选

YouTube Data API v3 → 监控频道上传 → RSS/Playlist → 去重过滤 → 人工确认

GET https://www.googleapis.com/youtube/v3/playlistItems?playlistId=UPLOADS&key=API_KEY
↓
2
B站交叉检索

B站搜索 API / 手动搜索 → 匹配标题+UP主 → 打标签（已有/部分/未找到）

GET https://api.bilibili.com/x/web-interface/search/all/v2?keyword={title}
↓
3
YouTube 下载（yt-dlp）

下载视频 + 英文字幕（自动字幕转 SRT）。支持 --download-sections 截取片段测试

python pipeline.py download "https://www.youtube.com/watch?v=VIDEO_ID"

# 测试用：只下载 2 分钟片段（--download-sections）
yt-dlp --download-sections "*35:00-37:00" \
  -f "bestvideo[height<=1080]+bestaudio/best[height<=1080]" \
  --write-auto-subs --sub-langs en --convert-subs srt \
  -o "_runtime/downloads/test_2min.%(ext)s" "URL"
↓
4
字幕提取 / 语音转文字（备用）

有字幕 → 直接用。无字幕 → faster-whisper 转录

faster-whisper video.mp4 \
  --model large-v3 --language en \
  --output_format srt
↓
5
AI 翻译（DeepSeek API）

逐句翻译英文 SRT → 中英双语 SRT（--- 分隔）。批次翻译 20 句/次，[N] 标记解析

python pipeline.py translate _runtime/downloads/video.en.srt

# 内部调用 DeepSeek API，批次翻译
# 输出: video.en_zh.srt（EN\n---\nZH 格式）
↓
6
字幕渲染（FFmpeg + ASS 烧录）

B站标准双语字幕：思源黑体 · ZH 42px(上)/EN 30px(下) · 黑色描边 · 10%边距

python pipeline.py render _runtime/downloads/video.webm _runtime/downloads/video.en_zh.ass

# internal: srt → ass (B站标准样式) → ffmpeg burn
ffmpeg -i video.webm -vf "ass=video.en_zh.ass" \
  -c:v libx264 -crf 23 -preset medium -c:a copy -y output.mp4
↓
7
B站上传打包

自动生成封面（截图+标题叠加）+ 元数据 JSON（标题/简介/标签）→ 一键上传

python pipeline.py bilibili output/video_subbed.mp4 \
  --title-cn "Claude Code 东京 2026 — Cat Wu 主题演讲（双语字幕）" \
  --title-en "Code with Claude Tokyo — Cat Wu Keynote" \
  --tags "Claude Code,Anthropic,AI编程,开发者大会" \
  --source-url "https://www.youtube.com/watch?v=..."

⚖️ 8. B站版权与合规专篇

⚠️ 声明 本文档基于公开信息整理，不构成法律建议。具体决策请咨询专业律师。

8.1 版权核心原理：你的翻译到底算什么？

在法律框架下，我们把 YouTube 视频下载、翻译、压制字幕、上传 B站 这个行为链拆开来看：

行为法律定性风险
下载 YouTube 视频 复制权（Reproduction） YouTube ToS 禁止下载，但个人使用极少被追究
翻译字幕（人工/AI） 翻译权（Translation / 演绎权） 翻译本身是独创性劳动，受著作权法保护
压制字幕到视频 改编权（Adaptation） 合成新载体，叠加了你的智力成果
上传到 B站 信息网络传播权 这是风险集中点 — 公开传播需要权利人许可

关键变量不是"你是否侵权"（技术上你确实复制和传播了他人的作品），而是：权利人会不会追究？平台会不会下架？

8.2 为什么技术大会 Keynote 比影视综艺安全得多？

不同内容类型的版权风险差异巨大。我们做的是海外科技公司的开发者大会、播客访谈、产品发布会——这跟搬运动漫、影视、综艺的法律处境完全不同。

内容类型版权方态度下架风险B站历史案例
影视剧/动漫/综艺 版权方有专业法务团队，主动监测下架 极高 2022年 TV Tokyo 投诉 → B站大量日本综艺下架；字幕组一夜清空
音乐/MV/演唱会 唱片公司有 Content ID 自动扫描 高 BGM 侵权是最常见的下架原因之一
新闻/时政/纪录片 涉及政治敏感，平台主动审核 高 涉及中国的政治话题几乎必然被拒
科技公司发布会/Keynote 公司希望传播，极少投诉 低 Apple WWDC、Google I/O、OpenAI DevDay 等大量翻译版在B站存活数年
技术播客/访谈 创作者通常欢迎扩大受众 低 Lex Fridman、Lenny's Podcast 翻译版广泛存在
产品 Demo/教程 本质是营销物料 极低 几乎没有下架案例
💡 核心逻辑：Anthropic、OpenAI、Google 等公司举办开发者大会的目的是传播技术理念、吸引开发者生态。他们主动把视频上传到 YouTube 并允许嵌入。翻译成中文传到 B站，本质上是免费帮他们做中国市场的本地化推广——这跟盗版电影的逻辑完全不同。但不代表零风险，仍需做足功课。

8.3 中国法律下的"合理使用"门槛

《著作权法》第24条列出了13种"合理使用"情形。对我们最可能适用的是：

条款内容适用性
第(六)项 为学校课堂教学或者科学研究，翻译或者少量复制已经发表的作品 不适用 — 面向公众的B站上传不属于"课堂教学"
第(八)项 图书馆、档案馆等为陈列或者保存版本的需要，复制本馆收藏的作品 不适用
第(二)项 为介绍、评论某一作品或者说明某一问题，在作品中适当引用他人已经发表的作品 部分适用 — 需要以"介绍/评论"为主，而非完整搬运

现实判断标准（来自上海知识产权法院 2023 年判决）：是否构成合理使用，核心看三点——

  1. 使用目的与性质：是否具有转换性（transformative）——添加了新价值、新功能、新表达
  2. 使用比例：是否超出了介绍/评论所需的必要范围
  3. 市场替代效应：是否会替代原作品的市场（中文翻译在B站 vs 英文原版在YouTube，市场重叠度低）
⚠️ 关键结论：完整搬运 + 纯翻译字幕 ≠ 合理使用。2025年长沙开福区法院判例（北京爱奇艺 v. B站）：字幕翻译仅是"同义转换"，不产生新的文学艺术价值，不构成合理使用。但如果在此基础上加入解说、点评、剪辑、分析，则有可能进入合理使用范畴。这也是为什么我们建议"翻译+解说"而非纯搬运。

8.9 常见误区澄清

误区真相
"标注'侵删'就安全了" 错误。"侵删"是社区惯例，不是法律抗辩。版权方仍可投诉。它的实际作用是向版权方表达善意，降低起诉概率。
"YouTube能看的就是公开的，可以随便搬" 错误。公开可见 ≠ 放弃版权。YouTube标准许可 ≠ CC许可。只有明确标注Creative Commons的视频可以自由使用。
"加了字幕就可以投自制" 错误。B站明确：未经版权方授权自行添加翻译字幕，一般投"转载"。除非有实质性剪辑、解说、配音等额外创作。
"AI翻译的字幕，不构成二创" 部分正确。纯AI翻译确实难以构成法律意义上的"二创"。但AI翻译+人工校对+精心排版+注释说明，整体构成可辨识的二次创作。
"科技公司不会追究个人" 通常对，但不绝对。大公司确实很少起诉个人翻译者，但不是零风险。如果视频爆火或涉及敏感解读，版权方可能介入。
"B站审核不过是因为内容问题，不是版权" 两种情况都有。B站审核有两套系统：内容安全审核 + 版权审核。有时稿件被退回是版权原因，但不会明确告知。

8.11 如果被下架/投诉了怎么办？

场景应对
B站审核退回 查看退回原因（通常有提示），修改后重新提交。常见原因：标题敏感词、BGM版权、缺少标签
版权方直接投诉 立即下架视频，回复版权方道歉并说明情况。询问是否可以合作或获得授权
被其他UP主举报"抄袭" 提供翻译过程证据（SRT文件、翻译时间戳）。字幕翻译+发布时间可证明独立创作
账号被扣分/限流 查看扣分原因，整改已发布内容。多次违规可能封号，尽早建立合规流程
🔑 一句话总结
技术内容的翻译搬运，在B站上属于"灰色但低风险"地带。核心策略不是追求零风险（做不到），而是：
做足善意姿态（授权请求+标注来源+不盈利+侵删声明）+ 避免触碰红线（政治/BGM/转载投自制）+ 做好下架预案。
